📋 核心機制區

四大自訂機制 • Memory Import • 工作流程
1️⃣ Slash Commands
快速執行預定義操作
  • 位置：.bob/commands/ 或 ~/.bob/commands/
  • 觸發：手動輸入 /command
  • 成本：極低 (50-200 tokens)
  • 內建命令：/init, /review, /create-pr
💡 最佳用途：常見任務快捷方式，按需載入。支援 argument-hint 參數提示
2️⃣ Custom Rules
定義「如何做事」的偏好設定
  • 位置：.bobrules 或 .bob/rules/
  • 觸發：自動載入
  • 成本：中等 (500-5,000 tokens)
  • 範圍：全域 + 工作區 + 模式特定
💡 最佳用途：編碼風格、技術規則、個人偏好
3️⃣ Skills
定義「做什麼」的工作流程配方
  • 位置：.bob/skills/*/SKILL.md 或 ~/.bob/skills/
  • 觸發：任務相關時提示（預設需批准）
  • 成本：較高 (1,000-10,000 tokens)
  • 限制：⚠️ 僅 Advanced 模式可用
  • 必要欄位：name, description（影響Bob判斷）
  • 支援檔案：可包含 checklist.md、templates、scripts 等
⚠️ 注意：專案級優先於全域級。可在設定中關閉批准提示。每次對話只載入一次
💡 管理指令：使用 npx skills CLI 工具管理 Skills（詳見下方「Skills 管理指令」）
4️⃣ Agents (AGENTS.md)
定義「為什麼」的團隊標準
  • 位置：專案根目錄 AGENTS.md
  • 觸發：自動載入（預設啟用）
  • 成本：高 (2,000-20,000 tokens)
  • 範圍：僅工作區
💡 最佳用途：團隊開發規範、核心原則

📚 Memory Import 記憶檔案匯入機制

🎯 核心概念

Memory Import 允許在 AGENTS.md 中使用 @path/to/file.md 語法引用外部檔案，實現模組化管理。
範例：@./.bob/memory/core/tools.md

✅ 優勢與安全機制

  • 模組化 拆分大型文件為小模組
  • 可維護 獨立更新各模組
  • 可重用 跨專案共享配置
  • 協作友善 團隊分工明確
  • 安全 循環引用偵測、檔案存取驗證、最大深度5層

💰 Token 節省機制

為何節省？
避免在 AGENTS.md、Rules、Skills 中重複相同內容，透過 @path 引用共享模組
運作方式：
每次新對話時，Bob 自動解析 @ 語法，動態載入引用的檔案內容到上下文
實際效益：
超過 200行 AGENTS.md 拆成多個模組，只載入需要的部分，可節省 50-70% token
⚠️ 錯誤處理
檔案不存在：
顯示錯誤訊息，不中斷載入
權限錯誤：
跳過該檔案，記錄警告
循環引用：
自動偵測並阻止無限迴圈
🔍 進階功能
• 程式碼區塊偵測（忽略程式碼中的@引用）
• 引用樹視覺化
• 巢狀引用支援
💡 最佳實踐
將超過 200行的 AGENTS.md 拆分成模組化檔案，每個檔案專注單一主題。詳見「建議結構」分頁

🛡️ Sandbox 環境隔離（Bob Shell）

可將 Bob Shell 操作隔離在安全的環境中，保護您的主機系統免受潛在危險操作的影響。
兩種方法：
macOS Seatbelt：sandbox-exec（僅macOS）
Container：Docker/Podman（跨平台）
三種配置：
1. 命令列：bob -s "cmd"
2. 環境變數：BOB_SHELL_SANDBOX=true
3. 設定檔：settings.json
Seatbelt Profiles：
permissive-closed：阻網路、專案寫入
permissive-proxied：代理、企業環境
restrictive-open：允許網路、嚴格檔案
restrictive-closed：最高安全性
⚠️ 限制：僅專案目錄、GUI無法運作、可能缺工具

📁 IBM Bob folder mapping（重新整理版）
設計重點：參考附圖的「folder fully mapped」呈現方式，將第 2 分頁改為 根目錄 → 全域設定 → 專案內 .bob/ 子系統 → 載入行為 的映射說明，讓使用者一眼看懂「放哪裡、何時生效、會不會進上下文」。
🧭 先看總原則
  • 專案根目錄：AGENTS.md 放「為什麼」、.bobignore 放「不能碰什麼」
  • 專案內 .bob/：放可操作的配置，例如 Rules、Commands、Skills、Memory Import
  • 全域 ~/.bob/：放跨專案共用設定，例如個人模式、MCP、常用規則
  • 高成本項：AGENTS.md、custom_modes.yaml、mcp_settings.json
  • 按需載入項：Slash Commands、Skills
🎯 三種判讀方式
高 會直接影響系統提示或長期上下文
中 規則類檔案，依載入順序生效
低 僅在觸發時載入，不會常駐
判讀公式：位置決定範圍、檔名決定用途、載入時機決定 token 成本。

檔案 角色 重點

路徑 主要用途 載入時機 建議寫法

settings.json 設定與行為開關 通常不進上下文 視為執行設定，不要混規範內容

🎯 使用流程：AGENTS.md 定義「為什麼」→ Rules 定義「如何做」→ Commands 快速執行 → Skills 處理複雜任務 → Custom Mode 切換專家角色 → Memory Import 提供詳細文件
💡 實戰技巧
  • 從小開始：先建立 AGENTS.md + 1-2 個 Rules
  • 逐步擴展：需要時才加 Commands 和 Skills
  • 保持簡短：每個檔案專注單一主題
  • 定期審查：移除不再使用的配置
  • 團隊協作：配置納入版本控制
⚠️ 常見錯誤
  • 過度配置：一開始就建立所有機制
  • 內容重複：在多個地方寫相同規則
  • 檔案過大：AGENTS.md 超過 500 行
  • 缺乏組織：Memory 檔案沒有分類
  • 忽略成本：載入過多不必要的內容
🚀 進階應用
  • 多專案共用：全域 Rules 放個人偏好
  • 模式專屬：rules-code/ 只在 Code 模式
  • 動態載入：Skills 按需批准載入
  • 分層管理：Memory 按主題分目錄
  • 版本演進：配置隨專案成長調整
