📋 核心機制區

四大自訂機制 • Memory Import • 工作流程
1️⃣ Slash Commands
快速執行預定義操作
  • 位置：.bob/commands/
  • 觸發：手動輸入 /command
  • 成本：極低 (50-200 tokens)
  • 範例：/review, /test, /deploy
💡 最佳用途：常見任務快捷方式，按需載入
2️⃣ Custom Rules
定義「如何做事」的偏好設定
  • 位置：.bobrules 或 .bob/rules/
  • 觸發：自動載入
  • 成本：中等 (500-5,000 tokens)
  • 範圍：全域 + 工作區 + 模式特定
💡 最佳用途：編碼風格、技術規則、個人偏好
3️⃣ Skills
定義「做什麼」的工作流程配方
  • 位置：.bob/skills/*/SKILL.md
  • 觸發：任務相關時提示（需批准）
  • 成本：較高 (1,000-10,000 tokens)
  • 限制：僅 Advanced 模式
⚠️ 注意：可包含支援檔案，使用者可控制載入
4️⃣ Agents (AGENTS.md)
定義「為什麼」的團隊標準
  • 位置：專案根目錄 AGENTS.md
  • 觸發：自動載入（預設啟用）
  • 成本：高 (2,000-20,000 tokens)
  • 範圍：僅工作區
💡 最佳用途：團隊開發規範、核心原則

• 大型專案：全套機制 + Skills
⚠️ 避免重複：
四大機制會累加 token 成本，避免在不同機制中重複相同內容，善用 Memory Import 共享配置
📚 Memory Import 記憶檔案匯入機制

🎯 核心概念

Memory Import 允許在 AGENTS.md 中使用 @path/to/file.md 語法引用外部檔案，實現模組化管理。

範例：@./.bob/memory/core/tools.md

✅ 優勢

  • 模組化拆分大型文件為小模組
  • 可維護獨立更新各模組
  • 可重用跨專案共享配置
  • 協作友善團隊分工明確
💰 Token 節省機制：
為何節省？避免在 AGENTS.md、Rules、Skills 中重複相同內容，透過 @path 引用共享模組
運作方式：每次新對話時，Bob 自動解析 @ 語法，動態載入引用的檔案內容到上下文
實際效益：超過 200行(過多行數) AGENTS.md 拆成多個不同模組，只載入需要的部分，可節省 50-70% token
💡 最佳實踐：將超過 200行(過多行數)的 AGENTS.md 拆分成多個不同模組化檔案，每個檔案專注單一主題，提高可維護性和協作效率。詳細結構請參考「建議結構」分頁。

💰 Token 節省策略

🎯 核心策略

  • 精準任務明確具體的描述
  • Line Range只讀需要的行
  • 批次處理一次完成相關任務
  • 選對工具apply_diff > write_to_file

⚠️ 高成本警示

  • 大檔案讀取>1000行檔案
  • 重複操作相同檔案多次操作
  • 模糊任務不明確的描述
  • 全專案掃描無限制搜尋
💡 黃金法則：明確具體 → 精準讀取 → 批次處理 → 選對工具 → 避免重複 → 簡潔溝通

🛡️ Sandbox 環境隔離（Bob Shell）

可將 Bob Shell 操作隔離在安全的環境中，保護您的主機系統免受潛在危險操作的影響。
兩種方法：
macOS Seatbelt：sandbox-exec（僅macOS）
Container：Docker/Podman（跨平台）
三種配置：
1. 命令列：bob -s "cmd"
2. 環境變數：BOB_SHELL_SANDBOX=true
3. 設定檔：settings.json
Seatbelt Profiles：
permissive-closed：阻網路、專案寫入
permissive-proxied：代理、企業環境
restrictive-open：允許網路、嚴格檔案
restrictive-closed：最高安全性
⚠️ 限制：僅專案目錄、GUI無法運作、可能缺工具

📁 IBM Bob folder mapping（重新整理版）
設計重點：參考附圖的「folder fully mapped」呈現方式，將第 2 分頁改為 根目錄 → 全域設定 → 專案內 .bob/ 子系統 → 載入行為 的映射說明，讓使用者一眼看懂「放哪裡、何時生效、會不會進上下文」。
🧭 先看總原則
  • 專案根目錄：AGENTS.md 放「為什麼」、.bobignore 放「不能碰什麼」
  • 專案內 .bob/：放可操作的配置，例如 Rules、Commands、Skills、Memory Import
  • 全域 ~/.bob/：放跨專案共用設定，例如個人模式、MCP、常用規則
  • 高成本項：AGENTS.md、custom_modes.yaml、mcp_settings.json
  • 按需載入項：Slash Commands、Skills
🎯 三種判讀方式
高 會直接影響系統提示或長期上下文
中 規則類檔案，依載入順序生效
低 僅在觸發時載入，不會常駐
判讀公式：位置決定範圍、檔名決定用途、載入時機決定 token 成本。

📦 根目錄層：最先理解的 3 個檔案
檔案 角色 重點
AGENTS.md 團隊規範、核心原則、為什麼這樣做 高 自動載入，避免寫成百科全書
.bobignore 限制 Bob 工具可接觸的檔案範圍 安全邊界，不是上下文內容
.gitignore 版本控制忽略規則 常與 .bobignore 一起維護
⚠️ 建議：AGENTS.md 留原則與索引，細節拆到 .bob/memory/，避免持續膨脹。
🌐 全域層 vs 專案層
層級適用範圍典型內容
~/.bob/所有專案個人偏好、常用模式、共用 MCP
{project}/.bob/單一專案專案規則、專案命令、專案 skills
優先級：專案設定 > 全域設定；模式專屬規則 > 共用規則。
🧩 `.bob/` 子資料夾用途地圖
路徑 主要用途 載入時機 建議寫法
rules/ 所有模式共用規則 中 對話開始時納入規則鏈 短、小、單一主題、多檔拆分
rules-{mode}/ 某一 mode 專用規則 中 僅該模式啟用時 只寫該 mode 真正需要的限制
commands/ Slash Commands 低 手動觸發時 一個命令只做一件事
skills/ 複雜任務流程、配方化執行 高 被選中時載入 SKILL.md 保持聚焦，支援檔案外拆
memory/ 被 @path 引用的模組化知識 由 AGENTS.md / 規則引用時 按主題分層，不要堆成單一大檔
custom_modes.yaml 定義角色與 custom instructions 高 進系統提示 只保留必要 persona 與行為
mcp_settings.json MCP server 註冊與描述 高 description 成本明顯 只連必要 MCP，description 要短
settings.json 設定與行為開關 通常不進上下文 視為執行設定，不要混規範內容
