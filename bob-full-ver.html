03 · Memory Import
📚
在 AGENTS.md 中使用 @path/to/file.md 語法引用外部檔案，實現模組化管理。
特性說明
模組化拆分大型文件為小模組
可重用跨專案共享配置
安全循環引用偵測、最大深度 5 層
效能節省 50–70% tokens
Token 節省機制
為何節省避免在 AGENTS.md、Rules、Skills 中重複相同內容，透過 @path 共享模組
運作方式新對話時自動解析 @ 語法，動態載入引用檔案到上下文
實際效益200+ 行 AGENTS.md 拆分後，只載入需要的部分
錯誤處理不存在顯示錯誤不中斷；自動偵測循環引用

05 · Token 節省策略
核心策略 vs 高成本警示
核心策略
  • 精準任務：明確具體的描述
  • Line Range：只讀需要的行
  • 批次處理：一次完成相關任務
  • 選對工具：apply_diff > write_file
高成本警示
  • 高 讀取 >1000 行大檔案
  • 高 相同檔案重複操作
  • 中 不明確的模糊任務
  • 中 無限制全專案掃描
成本估算參考
操作類型Token 成本建議
簡單查詢100–500直接執行
檔案修改500–2,000apply_diff
複雜重構2,000–10,000分階段處理
全專案分析10,000+精準指定範圍
⚠️
以上僅供參考，依實際使用為主。連接 MCP 會增加基礎成本，只在必要時使用。
06 · v2 全新功能
🤖 Subagents — 平行子代理
ℹ️
讓 Bob 生成獨立代理處理聚焦任務，避免污染主對話上下文。
類型能力適用
explore唯讀、輕量模型搜尋程式碼
general完整工具、預設模型讀寫獨立任務
  • Bob 決定生成子代理 → 你批准 → 子代理執行
  • 子代理有獨立 context window，不共享
  • 任務完成後摘要回傳主對話
💡
vs Subtask：Subagent 靜默執行；Subtask 在 UI 顯示完整對話可追蹤。
⏪ Rollback — 快照還原
ℹ️
每個任務前自動建立快照，底層使用 Shadow Git Repository。
快照時機任務開始時、檔案修改前
兩大功能View diff 比較差異   Rollback 還原
還原選項Restore files：僅還原檔案，保留對話
Restore files & task：還原並刪後續對話（不可逆）
限制僅追蹤 Bob 任務內的變更，.gitignore 排除的不追蹤
💡
Bob 輸出變差時，直接 Rollback 重試比繼續糾正更有效！
✏️ Literate Coding
  • 啟動：Cmd+I / Ctrl+I
  • 生成：Cmd+Enter 觸發 inline diff
  • 接受：Cmd+Enter
  • 拒絕：Cmd+Shift+Backspace
💡
不需切換 Chat，原地描述意圖即可生成。

高

中高

建議：僅受控環境下啟用自動批准
🛡️ 企業級六層防護（客戶 PoC 必備）
1️⃣.bobignore排除客戶資料（語法同 .gitignore）
2️⃣信任資料夾限制存取範圍（Shell 專用）
3️⃣禁用自動批准所有操作需確認
4️⃣唯讀 API Key限制 AI 權限
5️⃣Sandbox 隔離容器或 macOS Seatbelt
6️⃣稽核監控記錄所有操作
💡

總原則
🧭 放哪裡
  • 專案根目錄：AGENTS.md 放「為什麼」、.bobignore 放「不能碰什麼」
  • 專案內 .bob/：放可操作的配置（Rules、Commands、Skills、Memory）
  • 全域 ~/.bob/：放跨專案共用設定（個人模式、MCP、常用規則）
  • 高成本項：AGENTS.md、custom_modes.yaml、mcp_settings.json
  • 按需載入項：Slash Commands、Skills
🎯 判讀公式
高直接影響系統提示或長期上下文
中規則類檔案，依載入順序生效
低僅在觸發時載入，不會常駐
🗝️
判讀公式：位置決定範圍、檔名決定用途、載入時機決定 token 成本
Canonical Folder Mapping
your-project/                              ← 工作區 / project root
├── AGENTS.md                              ← 團隊原則、開發標準（自動載入）
├── .bobignore                             ← 阻擋敏感檔案/資料夾
├── .gitignore                             ← 常與 .bobignore 搭配
└── .bob/                                  ← 專案級 Bob 設定，優先於全域
    ├── settings.json                      ← 專案設定；不直接進上下文
    ├── custom_modes.yaml                  ← 自訂模式；模式指令進系統提示
    ├── mcp_settings.json                  ← MCP 設定；description 成本高
    ├── rules/         *.md               ← 所有模式共用；持續影響回應
    ├── rules-{mode-slug}/  *.md          ← 只在該 mode 載入
    ├── commands/      *.md               ← 手動觸發時載入，低成本
    ├── skills/
    │   └── skill-name/
    │       ├── SKILL.md                   ← 技能主定義；選擇時載入
    │       └── support-files...
    └── memory/                            ← Memory Import 模組庫
        ├── intro/   core/   features/
        ├── security/   best-practices/
        └── project-context.md

~/.bob/                                    ← 全域設定，跨專案共用
├── settings.json   custom_modes.yaml
├── mcp_settings.json
└── rules/  rules-{mode}/  commands/  skills/
.bob/ 子資料夾用途地圖
路徑主要用途載入時機建議寫法
rules/所有模式共用規則中 對話開始時短、小、單一主題、多檔拆分
rules-{mode}/某一 mode 專用規則中 僅該模式啟用時只寫該 mode 真正需要的限制
commands/Slash Commands低 手動觸發時一個命令只做一件事
skills/複雜任務流程高 被選中時載入SKILL.md 聚焦，支援檔外拆
memory/@path 引用的模組化知識由 AGENTS.md 引用時按主題分層，不要堆成大檔
custom_modes.yaml定義角色與 custom instructions高 進系統提示只保留必要 persona
mcp_settings.jsonMCP server 註冊與描述高 description 成本明顯只連必要 MCP，description 要短
settings.json設定與行為開關通常不進上下文視為執行設定，不要混規範

🔄 載入順序 & 實務建議
規則鏈（低 → 高）
  • ① 全域規則 ~/.bob/rules/
  • ② 工作區模式規則 .bob/rules-{mode}/
  • ③ AGENTS.md（專案根目錄）
  • ④ 工作區一般規則 .bob/rules/
按需載入
  • Commands：使用該命令時才載入
  • Skills：被挑選/批准後才載入
  • MCP：連線時 description 進提示
實務建議
  • 長內容放 Memory，不放根 AGENTS
  • 共用規則短、模式規則更短
  • 高成本設定只保留必要項
